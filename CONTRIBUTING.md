# Beitragen zum Critical Pedagogy Skill

Vielen Dank für dein Interesse, zu diesem Projekt beizutragen! Dieses Dokument beschreibt, wie du Verbesserungen vorschlagen und einreichen kannst.

## Wie du beitragen kannst

### Fehler melden oder Verbesserungen vorschlagen

1. Erstelle ein [Issue](https://github.com/malkreide/critical-pedagogy-skill/issues) mit einer klaren Beschreibung
2. Verwende aussagekräftige Titel
3. Beschreibe den aktuellen Zustand und den gewünschten Zustand

### Änderungen einreichen

1. Forke dieses Repository
2. Erstelle einen Feature-Branch: `git checkout -b feature/meine-verbesserung`
3. Nimm deine Änderungen vor
4. Committe mit einer klaren Nachricht: `git commit -m "Beschreibung der Änderung"`
5. Pushe den Branch: `git push origin feature/meine-verbesserung`
6. Erstelle einen Pull Request

## Richtlinien

### Sprache und Stil

- **Schweizerische Rechtschreibung** verwenden (kein ß, «Guillemets» statt „Anführungszeichen")
- Gendergerechte Sprache wo möglich
- Fachbegriffe bei erster Verwendung kurz erklären
- Ton: kritisch, aber konstruktiv und respektvoll

### Inhaltliche Qualität

- Theoretische Bezüge korrekt zitieren und einordnen
- Neue Analyseschritte oder -techniken mit Beispielen illustrieren
- Bei Rechtsverweisen: Rechtsstand prüfen und Geltungsbereich angeben
- Schweizer Bildungskontext berücksichtigen, aber Übertragbarkeit auf andere Kontexte ermöglichen

### Technische Anforderungen

- `SKILL.md` muss gültiges Markdown sein
- YAML-Frontmatter in `SKILL.md` muss korrekt formatiert sein
- Änderungen an der `description` im Frontmatter sollten die Trigger-Erkennung verbessern

## Verhaltenskodex

Wir erwarten von allen Beitragenden einen respektvollen und konstruktiven Umgang. Diskriminierung, Herabsetzung oder aggressive Kommunikation werden nicht toleriert.

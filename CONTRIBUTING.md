Thank you for your interest in contributing to this project! This document describes how you can suggest and submit improvements.

## How to Contribute

### Report Issues or Suggest Improvements

1. Create an [Issue](https://github.com/malkreide/critical-pedagogy-skill/issues) with a clear description
2. Use descriptive titles
3. Describe the current state and the desired state

### Submit Changes

1. Fork this repository
2. Create a feature branch: `git checkout -b feature/my-improvement`
3. Make your changes
4. Commit with a clear message: `git commit -m "Description of change"`
5. Push the branch: `git push origin feature/my-improvement`
6. Create a Pull Request

### Content Quality

- For legal references: verify legal status and specify jurisdiction
- Consider the Swiss educational context, but enable transferability to other contexts

### Technical Requirements

- `SKILL.md` and `SKILL_EN.md` must be valid Markdown
- YAML frontmatter in skill files must be correctly formatted
- Changes to the `description` in the frontmatter should improve trigger recognition

## Areas for Contribution

We are particularly interested in contributions in these areas:

- **Localization:** Adaptations for other educational systems (with appropriate legal frameworks)
- **Domain-specific extensions:** Additions for specific subjects or professions
- **Example dialogues:** Effective critical-pedagogical analyses of real-world documents
- **Evaluation metrics:** Methods for measuring analysis quality
- **Integration guides:** Instructions for various platforms

## Code of Conduct

We expect all contributors to engage respectfully and constructively. Discrimination, belittlement, or aggressive communication will not be tolerated.
